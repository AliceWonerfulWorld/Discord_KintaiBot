# 🧭 機能一覧テンプレート（自作勤怠システム）

---

## 0️⃣ 前提定義（最初に決める）

| 項目 | 内容 |
| :--- | :--- |
| **対象ユーザー** | Discordを日常的なコミュニケーションツールとして利用しているチーム・コミュニティ、または個人 |
| **提供価値** | Discord上で完結する手軽な打刻体験と、Web画面による柔軟な勤怠管理（完全無料でカスタマイズ可能） |
| **コア体験** | スラッシュコマンド（`/kintai`）によるシームレスな打刻と、管理画面での直感的な打刻修正 |
| **成功指標** | 日々の打刻の定着（打刻忘れの減少）、および月末の勤怠集計・修正にかかる手間の最小化 |

---

# 1️⃣ 機能カテゴリ設計

## 【カテゴリA：Discord Bot機能（打刻・通知）】

| # | 機能名 | 概要 | ユーザー価値 | 優先度 | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| A-1 | 出勤打刻コマンド | `/kintai 開始` で出勤時間を記録 | Discordを開いてすぐに業務を開始できる | 🔴 P0 | `attendances` テーブルにInsert |
| A-2 | 退勤打刻コマンド | `/kintai 終了` で退勤時間を記録 | 業務終了をスムーズに記録・報告できる | 🔴 P0 | `attendances` テーブルをUpdate |
| A-3 | 休憩打刻コマンド | `/kintai 休憩` で休憩の開始/終了を記録 | 1日に複数回の休憩を正確に記録できる | 🔴 P0 | `breaks` テーブルにInsert/Update |
| A-4 | ステータス確認コマンド | `/kintai 状況` で現在の自分の打刻状態を表示 | 今自分が「出勤中」か「休憩中」か把握できる | 🟡 P1 | Botの返信メッセージで現在の状態を返す |
| A-5 | 打刻忘れリマインド | 退勤せずに日を跨いだ場合などにDMやメンションで通知 | 打刻忘れを早期に発見できる | 🟢 P2 | 定期実行（Cron）処理が必要 |

---

## 【カテゴリB：Web管理画面（閲覧・編集）】

| # | 機能名 | 概要 | ユーザー価値 | 優先度 | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| B-1 | 勤怠データ一覧表示 | ユーザー別・月別で勤怠記録と合計労働時間を一覧表示 | チーム全員の稼働状況をブラウザでサクッと確認できる | 🔴 P0 | Next.jsで実装。Supabaseからデータ取得 |
| B-2 | 出退勤の編集・追加 | 打刻漏れや時間の誤りをUI上から直接修正・新規追加する | コマンドを間違えても簡単にリカバーできる | 🔴 P0 | 管理者のみ操作可能にするか要検討 |
| B-3 | 休憩時間の編集 | 休憩の開始・終了時間をUI上から修正する | 休憩の取り忘れや時間のズレを正確に直せる | 🟡 P1 | |
| B-4 | CSV/Excelエクスポート | 表示している月の勤怠データをCSV等でダウンロードする | 給与計算や外部システムへの連携が楽になる | 🟢 P2 | |

---

## 【カテゴリC：アカウント・システム管理】

| # | 機能名 | 概要 | ユーザー価値 | 優先度 | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| C-1 | Discord OAuthログイン | Web管理画面にDiscordアカウントでログインする | 新しくパスワードを管理する手間がない | 🔴 P0 | Supabase AuthのDiscord連携を利用 |
| C-2 | 権限管理（Role） | 管理者（Admin）と一般（User）で編集権限を分ける | 一般ユーザーが他人のデータを勝手に書き換えるのを防ぐ | 🟡 P1 | `users` テーブルの `role` カラムで判定 |
| C-3 | 深夜・日跨ぎ設定 | 深夜24時を過ぎた打刻を「前日の勤務」として扱う設定 | 徹夜作業や深夜シフトでも正しい営業日で集計できる | ⚪ P3 | 初期は手動修正でカバーし、後回しでも可 |

# 2️⃣ 優先順位の定義基準（共通ルール）

| 優先度 | 判断基準 |
| :--- | :--- |
| 🔴 P0 | これがないとプロダクトが成立しない（MVPに必要な最小限の機能） |
| 🟡 P1 | 体験を大きく向上させる（あると便利・実運用に耐えうる） |
| 🟢 P2 | 運用効率・継続率を改善する（自動化や外部連携など） |
| ⚪ P3 | 将来の拡張性・差別化（特殊なユースケースへの対応） |

# 3️⃣ フェーズ整理テンプレート

```text
【Phase 0 - MVP】（🔴 P0のみ実装）
・Discordでの出勤/退勤/休憩の打刻（A-1, A-2, A-3）

・日跨ぎ・深夜残業の自動判定（C-3）
